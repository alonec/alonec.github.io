java中的static关键字解析

发表于 2018-05-20 | 分类于 java | |
字数统计: 1,577 字 | 阅读时长 ≈ 6
static关键字的用途和常见笔试面试题的总结。

一、static关键字的用途

“static方法就是没有this的方法。在static方法内部不能调用非静态方法，反过来是可以的。而且可以在没有创建任何对象的前提下，仅仅通过类本身来调用static方法。这实际上正是static方法的主要用途。”

——《java编程思想》

被static关键字修饰的方法或者变量不需要依赖于对象来进行访问，只要类被加载了，就可以通过类名去进行访问。这是static最主要的作用。

static可以用来修饰类的成员方法、类的成员变量，另外可以编写static代码块来优化程序性能。

1.静态变量

static变量也称作静态变量，静态变量和非静态变量的区别是：静态变量被所有的对象所共享，在内存中只有一个副本，它当且仅当在类初次加载时会被初始化。而非静态变量是对象所拥有的，在创建对象的时候被初始化，存在多个副本，各个对象拥有的副本互不影响。

注意，java中规定，static只能修饰成员变量，而不能修饰局部变量。即方法中不能有static关键字存在。

static成员变量的初始化顺序按照定义的顺序进行初始化。

2.静态方法

由于静态方法不依赖于任何对象就可以进行访问，因此对于静态方法来说，是没有this的，因为它不依附于任何对象，既然都没有对象，就谈不上this了。并且由于这个特性，在静态方法中不能访问类的非静态成员变量和非静态成员方法，因为非静态成员方法/变量都是必须依赖具体的对象才能够被调用。

　　但是要注意的是，虽然在静态方法中不能访问非静态成员方法和非静态成员变量，但是在非静态成员方法中是可以访问静态成员方法/变量的。

3.静态块

static关键字还有一个比较好的作用就构建静态代码块以优化程序。static块可以置于类中的任何地方，类中可以有多个static块。在类初次被加载的时候，会按照static块的顺序来执行每个static块，并且只会执行一次。
